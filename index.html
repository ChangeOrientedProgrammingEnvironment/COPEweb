COPE - Change-Oriented Programming Environment

aims to make software changes first class citizens in the development environment

In order to address COPE project goals, we have created ClientRecorder plugin for Eclipse that records source code changes and unit-test runs. Data collected by ClientRecorder will allow us to understand how developers change complex code and use Test-Driven Development practices to structure source code edits.

If you use Eclipse for any of your projects, we ask you to install ClientRecorder plugin to help us test plugin. Plugin will record source code changes in your IDE and send collected data to OSU data storage. All you need to do is to install Eclipse plugin and report to cope@engr.orst.edu if you have any problems with the plugin (exceptions, errors, unstable Eclipse, etc). You can find plugin installation instructions here: COPE ClientRecorder installation instructions.
