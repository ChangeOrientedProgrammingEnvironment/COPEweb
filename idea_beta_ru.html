Всем всем всем Java-программистам, использующим IntelliJ IDEA для работы над личными проектами или проектами с открытым исходным кодом! У вас есть возможность сделать важный вклад в науку, не прилагая абсолютно никаких интеллектуальных усилий!

Группа исследователей из Университета штата Орегон, занимающаяся проблемами эволюции и поддержки ПО проводит исследование c целью анализа практик TDD, используемых профессиональными Java-разработчиками в повседневной работе и жизни. Нужна ваша помощь в бета-тестировании плагина для IntelliJ IDEA, созданным для сбора данных об изменениях исходного кода и запуске юнит-тестов в IntelliJ IDEA. COPE Recorder (так называется плагин для IntelliJ IDEA, разработанный исследователями) предназначен для записи изменений исходного кода, запуске и результатах юнит-тестов, выполненных при работе в интегрированной среде разработки IntelliJ IDEA. Плагин регулярно отправляет данные на сервер Университета штата Орегон для последующего анализа. Поэтому для участия в бета-тестировании приглашаются Java-разработчики, которым не нужно беспокоиться об интеллектуальной собственности написанного исходного кода.

Если вы – Java-разработчик, использующий IntelliJ IDEA и вы работаете над личным, учебным или open-source проектом, то всё, что вам нужно сделать – это:

  1. Ознакомиться с информированным согласием
  2. Установить COPE-плагин для IntelliJ IDEA нажав кнопку 'ACCEPT'
  3. Заполнить короткий опросник после установки плагина
  4. Продолжать писать на Java в обычном для вас режиме
Если вдруг вы столкнетесь с ошибками работы плагина, просьба отправить отчет с кратким описанием обнаруженной ошибки используя эту форму. После установки плагина вы автоматически становитесь участником исследования даже если у вас не будет возможности отправить отчет об ошибке. В любой момент вы можете прекратить участие в исследовании удалив плагин (инструкции по удалению плагина находятся здесь).

Если вы участвовали в исследовании (не удалили плагин) до даты окончания сбора данных (будет опубликована на странице исследовательского проекта после окончания бета-тестирования), а также предоставили электронный адрес для обратной связи при заполнении опросника, по окончании анализа данных вам будет отправлен детальный отчет с описанием практик TDD, которые вы использовали на протяжении периода исследования. Отчет будет содержать персонализированную статистику и метрики, собранные в результате анализа данных. В частности, отчет будет содержать информацию о персональном рейтинге относительно других участников исследования. Рейтинг будет составлен на основе критериев зрелости практик TDD, применямых каждым участником исследования.
